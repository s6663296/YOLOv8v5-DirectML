# AI AIMBOT MAX v1.0 完整安裝教學（DML / CUDA / TensorRT）

本教學整合本專案三種推理後端的完整安裝流程：

- DirectML (`dml`)：AMD / Intel / NVIDIA 都可用
- CUDA (`cuda`)：NVIDIA 專用
- TensorRT (`trt`)：NVIDIA 專用，需使用 `.engine` 模型

> 建議先完整看過一次再開始安裝。  
> 專案根目錄以下用 `D:\AIMBOT-MAX-v1.0` 示範，請依你的實際路徑替換。

---

## 1. 先決條件

### 1.1 作業系統與 Python

- Windows 10/11 64-bit
- Python 3.10.x（建議 3.10.8）

### 1.2 顯示卡與驅動

- DML：AMD / Intel / NVIDIA 皆可
- CUDA / TensorRT：NVIDIA 顯卡，建議 RTX 20 系列以上
- 安裝最新版 NVIDIA Driver（CUDA/TRT 必做）

### 1.3 必要工具

- Git（可選）
- Visual C++ Redistributable（若遇 DLL 問題可安裝專案內 `VC_redist.x64.exe`）

---

## 2. 後端差異與模型格式

| 後端 | 設定值 | 模型格式 | 適用 GPU |
|---|---|---|---|
| DirectML | `dml` | `.onnx` | AMD/Intel/NVIDIA |
| CUDA | `cuda` | `.onnx` | NVIDIA |
| TensorRT | `trt` | `.engine` | NVIDIA |
| CPU | `cpu` | `.onnx` | 全平台 |

重點：

- `dml/cuda/cpu` 走 ONNX Runtime（模型用 `.onnx`）
- `trt` 走 TensorRT Strategy（模型用 `.engine`）
- UI 內「推理後端」可直接切換：
  - `DirectML (AMD/Intel/NVIDIA)`
  - `CUDA (NVIDIA)`
  - `TensorRT (NVIDIA 高效能)`
  - `CPU (通用)`

---

## 3. 建議安裝策略（重要）

由於 ONNX Runtime 的 DirectML 與 CUDA 套件通常不建議混裝，**最穩定做法是分 3 個虛擬環境**：

- `venv_dml`：給 DML
- `venv_cuda`：給 CUDA
- `venv_trt`：給 TensorRT

你也可以只用一個 `venv`，但切換 `dml/cuda` 時要重裝 ONNX Runtime 套件，較容易混亂。

## 5. DirectML 模式安裝（dml）

### 5.1 安裝步驟

```cmd

python -m venv venv_dml
venv_dml\Scripts\activate.bat

pip install --upgrade pip
pip install -r requirements.txt
```

```cmd
python -c "import onnxruntime as ort; print('ORT:', ort.__version__); print(ort.get_available_providers())"
```

預期至少包含：

- `DmlExecutionProvider`
- `CPUExecutionProvider`


## 6. CUDA 模式安裝（cuda）

### 6.1 前置需求

- NVIDIA Driver 最新版
- 建議 CUDA Toolkit 12.x（與驅動相容）

### 6.2 安裝步驟

```cmd
cd /d "D:\AIMBOT-MAX-v1.0"
python -m venv venv_cuda
venv_cuda\Scripts\activate.bat

pip install --upgrade pip
pip install -r requirements.txt

pip uninstall onnxruntime-directml -y
pip install onnxruntime-gpu==1.23.0
```

### 6.3 驗證 CUDA provider

```cmd
python -c "import onnxruntime as ort; print('ORT:', ort.__version__); print(ort.get_available_providers())"
```

預期包含：

- `CUDAExecutionProvider`
- `CPUExecutionProvider`

### 6.4 啟動程式

```cmd
python main.py
```

UI 內選：`CUDA (NVIDIA)`，模型使用 `.onnx`。

## 7. TensorRT 模式安裝（trt）

### 7.1 前置需求

- NVIDIA Driver 最新版
- CUDA Toolkit 12.x
- TensorRT 10.x（Windows / CUDA 12.x 版本）

### 7.2 下載與環境變數

1. 下載 TensorRT 10.x，解壓至固定路徑（例）：`C:\TensorRT-10.13.3.9`
2. 系統環境變數 `Path` 新增：`C:\TensorRT-10.13.3.9\lib`
3. 重新開啟 CMD

### 7.3 安裝步驟

```cmd
cd /d "D:\AIMBOT-MAX-v1.0"
python -m venv venv_trt
venv_trt\Scripts\activate.bat

pip install --upgrade pip
pip install -r requirements.txt

pip install "C:\TensorRT-10.13.3.9\python\tensorrt-10.13.3.9-cp310-none-win_amd64.whl"
pip install cuda-python
```

### 7.4 驗證 TRT

```cmd
python -c "import tensorrt as trt; print('TensorRT:', trt.__version__)"
python -c "from cuda.bindings import runtime; print('cuda-python: OK')"
python -c "from Module.tensorrt_inference import is_tensorrt_available; print('TensorRT available:', is_tensorrt_available())"
```

## 8. 模型轉換（ONNX/PT -> TensorRT Engine）

### 8.1 啟動 GUI（建議）

```cmd
cd /d "D:\AIMBOT-MAX-v1.0"
venv_trt\Scripts\activate.bat
python trt_converter_gui.py
```

或使用批次檔：

```cmd
run_trt_converter_gui.bat
```

> 若你不是用預設 `venv`，建議先手動啟用對應環境再執行 `python trt_converter_gui.py`。

### 8.2 GUI 操作

1. 轉換模式選擇：
   - `ONNX -> TensorRT (.engine)`
   - `PyTorch (.pt) -> TensorRT (.engine)`
2. 選輸入模型
3. 設輸出 `.engine`
4. 選 FP16/FP32、Workspace、Verbose
5. 點「開始轉換」

### 8.3 PT 模式方法差異

- `direct`：ultralytics 直接輸出 engine（快）
- `onnx`：先轉 ONNX 再轉 TRT（相容性較高）

## 10. 常見問題排解

### 10.1 CUDA 模式選了但跑不起來

檢查：

```cmd
python -c "import onnxruntime as ort; print(ort.get_available_providers())"
```

若沒有 `CUDAExecutionProvider`：

- 確認在 `venv_cuda`
- 確認已安裝 `onnxruntime-gpu==1.23.0`
- 更新 NVIDIA Driver / CUDA

### 10.2 DML 模式不可用

若沒有 `DmlExecutionProvider`：

- 確認在 `venv_dml`
- 確認已安裝 `onnxruntime-directml==1.23.0`

### 10.3 TensorRT 顯示不可用

- 確認 `Path` 有 `C:\TensorRT-10.13.3.9\lib`
- 確認安裝 TRT wheel 與 `cuda-python`
- 重新開 CMD

### 10.4 TensorRT 模式使用 `.onnx` 失敗

TRT 模式必須使用 `.engine`，請先用 `trt_converter_gui.py` 轉換。

### 10.5 批次檔啟動失敗

- 先手動啟用環境再跑：`python trt_converter_gui.py`
- 確認目前路徑在專案根目錄
